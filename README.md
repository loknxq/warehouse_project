# README.md

## Система управления складом (Inventory Management System)

### Краткое описание проекта

Проект представляет собой программное обеспечение для автоматизации складского учёта. Приложение предназначено для малых и средних торговых предприятий. Оно позволяет оформлять приход и расход товаров, отслеживать актуальные остатки в реальном времени, формировать заказы поставщикам при достижении минимального порога запасов и генерировать аналитические отчёты. Система заменяет ручной учёт с помощью бумажных накладных и электронных таблиц.

### Основные функции

Система поддерживает четыре роли пользователей: кладовщик, менеджер по продажам, руководитель склада и администратор. Для каждой роли доступен свой набор функций.

Кладовщик может оформлять приход товара от поставщика, оформлять расход товара клиенту и проверять текущие остатки.

Менеджер по продажам может проверять остатки перед тем, как обещать товар клиенту, а также резервировать товар под конкретный заказ.

Руководитель склада может просматривать остатки, формировать отчёты по движению товаров и по востребованности позиций, а также создавать и утверждать заказы поставщикам. Система автоматически предлагает заказ, когда остаток опускается ниже минимального порога.

Администратор может управлять учётными записями пользователей и вести справочники товаров и поставщиков.

### Технологии и требования

Для сборки и работы проекта необходимо следующее программное обеспечение:

Система сборки CMake версии 3.10 или выше. Компилятор C++ с поддержкой стандарта C++17. Подойдут GCC версии 7 или новее, Clang версии 5 или новее, а также MSVC 2017 и новее.
Проект использует SQLite3 для хранения данных о товарах, операциях и пользователях. Также используется библиотека pthread для многопоточного логгирования. В операционных системах Linux и macOS pthread обычно уже установлена. Для Windows требуется подсистема Windows Subsystem for Linux или отдельная настройка pthread.

### Структура проекта

Файлы проекта организованы по логическим модулям. Корневая папка содержит файл CMakeLists.txt для настройки сборки, файл README.md с инструкцией и две основные папки: src и include.

Папка src содержит все исходные файлы с расширением .cpp. Внутри неё есть подпапки: core для основных сущностей (товар, остаток, поставщик, пользователь), documents для классов приходных и расходных операций, services для сервисов (логгирование, расчёт заказов, генерация отчётов) и strategies для классов алгоритмов расчёта рекомендаций.

Папка include содержит соответствующие заголовочные файлы с расширением .h. Структура подпапок в include повторяет структуру в src.

Папка build создаётся автоматически при сборке. В неё попадают объектные файлы и готовый исполняемый файл. Сама папка build не входит в репозиторий, она создаётся на машине пользователя.

### Инструкция по сборке

Сборка проекта выполняется из командной строки. Перед началом убедитесь, что установлены CMake, компилятор C++17 и библиотека SQLite3.

В дистрибутивах Linux на базе Debian или Ubuntu библиотеку SQLite3 можно установить командой через пакетный менеджер. Вместо точной команды здесь указано, что нужно установить пакеты libsqlite3-dev и build-essential.

В macOS библиотека SQLite3 уже присутствует в системе. Для установки CMake можно использовать Homebrew.

В Windows рекомендуется использовать MSYS2 или установить Visual Studio с поддержкой CMake.

После установки всех зависимостей откройте терминал или командную строку и перейдите в корневую папку проекта. Выполните первую команду для генерации файлов сборки. Эта команда указывает CMake взять исходный код из текущей папки и создать папку build с файлами сборки.

Затем выполните вторую команду для запуска самой сборки. CMake скомпилирует все исходные файлы и соберёт их в один исполняемый файл.

Если сборка прошла успешно, в папке build появится исполняемый файл с именем warehouse_app.

### Запуск проекта

Перейдите в папку build с помощью команды смены директории. Затем запустите исполняемый файл. В операционных системах Linux и macOS это делается через точку и слеш перед именем файла. В Windows достаточно ввести имя файла с расширением .exe.

После запуска программа покажет текстовое меню. Выберите нужную операцию, вводя соответствующий номер. Следуйте подсказкам на экране. Все изменения сохраняются в базе данных SQLite3, которая создаётся автоматически в папке с программой при первом запуске.

### Список использованных сторонних библиотек

Библиотека SQLite3 используется для хранения всех данных проекта. Это легковесная встраиваемая реляционная база данных. Она не требует отдельного сервера и хранит всё в одном файле.

Библиотека pthread используется для записи логов в отдельном потоке. Это позволяет не замедлять основные операции прихода и расхода товара.
Других внешних библиотек проект не использует. Все остальные компоненты реализованы на стандартном C++17.
